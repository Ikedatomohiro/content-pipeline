# ブログ記事テンプレート集

執筆担当（Agent 5）・構成担当（Agent 3）・校正担当（Agent 6）が参照するテンプレート。
記事の各パートごとに定型構成を提供し、効率的かつ読まれる記事を生成する。

---

## 記事の4大構成要素

すべてのブログ記事は以下の4要素で構成する:

1. **タイトル** — 読みたいと感じさせる目を引くもの
2. **リード文（導入文）** — 読者の共感を引き出し、どんな悩みが解決できるか明確に
3. **本文** — 論理的な見出し構成、読みやすさと分かりやすさを重視
4. **まとめ** — ここだけ読めば記事の内容がわかるように

---

## タイトルのテンプレート

### テンプレートA: 比較・解説型
```
{商品A}と{商品B}の違いを徹底比較！ {ターゲット}が選ぶべきなのはどっち？
```

### テンプレートB: おすすめ・ランキング型
```
{テーマ}のおすすめ{N}選！ {権威性を示す一言}が厳選して紹介
```

### タイトルのルール
- 記事内容を要約する
- 30〜40文字に収める
- 重要な要素はなるべく先に書く
- 読者の目を引く工夫をほどこす
- どんな悩みが解決するか分かりやすく書く

---

## リード文のテンプレート

### テンプレートA: 共感型（悩みの代弁）
```
「{悩み1}」や「{悩み2}」で悩んでいませんか？
私も長年{同じ悩み}なのでとても困っています。
```

### テンプレートB: 解決提示型
```
本記事では「{悩み1}」や「{悩み2}」の解決法を紹介します。
私も実際に使っており、{悩み}が嘘のように解消されました。
```

### リード文の推奨構成（AとBを組み合わせる）
1. 読者の悩みを代弁する（共感）
2. 解決方法を提示する
3. どんな人を対象とした記事かを示す
4. 記事を読むメリットを伝える

### リード文のルール
- 1スクロール内（1画面内）に収まる長さにする
- 短く簡潔にまとめる
- 悩みの代弁 → 解決方法の提示 → 対象読者の順で書く

---

## 本文のテンプレート

### テンプレート1: PREP法（万能型・基本）
```
「{悩み}」の悩みは「{解決策}」で解決できます。（結論）
なぜなら「{理由}」だからです。（理由）
具体的な手順は「{手順}」で進めてください。（具体例）
おすすめの「{選択肢}」を{N}つ紹介します。（具体例）
「{テーマ}」のメリットとデメリットをそれぞれ挙げてみましょう。（具体例）
ぜひ「{悩み}」で悩んだら「{解決策}」で解決しましょう。（結論）
```

### テンプレート2: 商品レビュー型
```
「{悩み}」で悩んでいませんか？（導入）
「{悩み}」には{商品A}がおすすめです（結論）
{商品A}の概要。おすすめポイントをざっくりと紹介（概要）
{商品A}のメリット（長所）をいくつか紹介（詳細）
{商品A}のデメリット（短所）も紹介。フォローも入れる（詳細）
{商品A}の口コミ・評判（口コミ）
{商品A}と競合商品の比較（比較）
{商品A}はこんな人におすすめ（タイプ別おすすめ）
{商品A}に関するよくある質問（Q&A）
「{悩み}」で悩んでいるなら{商品A}！（再び結論）
```

### テンプレート3: お悩み解決型
```
「{悩み}」で悩んでいませんか？（導入）
実は私も長年「{悩み}」で悩んでいました（共感）
私は「{解決策}」することで長年の悩みが解消されました（結論）
今は「{良い状態}」や「{良い状態}」になってとても幸せです（ベネフィット）
「{悩み}」の解決方法をいくつか紹介（詳細）
一番のおすすめは{商品A}を使う方法です（絞り込み）
{商品A}の具体的な詳細（マネタイズ）
{商品A}の競合比較・口コミ・Q&A（不安・疑問の解消）
「{悩み}」で悩んでいるなら{商品A}！（再び結論）
```

### 本文の見出しルール
- 見出しタグは正しい入れ子構造にする（h2 > h3 > h4）
- h1は記事タイトル用に予約、本文では使わない
- h2にはできる限りキーワードを含める
- h3は無理にキーワードを含めなくてよい

## まとめのテンプレート

### テンプレートA: 箇条書き要約型
```
本記事のまとめは以下のとおりです。
- 「{要点1}」は「{結論1}」で解決できる
- やり方は「{手順}」するだけ
- 「{注意点}」に注意するともっと効率的になる
```

### テンプレートB: アクション誘導型
```
以下のリンクで{関連テーマ}についての解説もしています。
ぜひあわせて読んでみてください。
```

### まとめのルール
- ブログの内容を簡潔に要約する
- 読者に具体的なアクションを促す
- 長く書きすぎない
- 関連記事やおすすめ商品への誘導を含める

---

## メタディスクリプションのテンプレート

```
本記事では{テーマ}の{悩み}を解決する方法を紹介します。
この記事を読むことで{得られる知識}がわかります。
もう{悩み}で悩むこともなくなるでしょう。
```

### メタディスクリプションのルール
- 120〜160文字で書く
- 記事の内容を簡潔に紹介する
- 記事を読むと解決できることを書く
- キーワードを自然に含める

---

## 読みやすい文章の7つのルール

1. **箇条書きを活用** — 3〜5個の項目数におさえ、短い文で端的に書く
2. **専門用語を減らす** — 小学生にやさしく説明するイメージで書く。使う場合は補足・注釈を入れる
3. **PREP法で構成** — 結論 → 理由 → 具体例 → 結論の順序で説得力を出す
4. **テンポの良い文章** — 一文は40〜60文字。3〜4文続いたら空行や箇条書きで区切る
5. **同じ語尾を連続させない** — 「です」「です」「です」は避ける
6. **冗長な表現を避ける** — 回りくどい言い回しは削る
7. **漢字・送り仮名・カタカナ表記を統一** — 表記ゆれをなくす

---

## SEOの基本ルール

- 1記事1キーワード（1記事1検索意図）
- タイトル・見出し・本文にキーワードを自然に含める
- キーワードの過剰使用は逆効果
- 検索意図を深く追求する（上位サイト分析・関連KW調査）
- E-E-A-T（経験・専門性・権威性・信頼性）を意識する

---

## 商品紹介記事のコツ

- その商品・サービスでどんな悩みが解決するか明確にする
- 商品を使ったあとの幸せな未来（ベネフィット）を見せる
- リアリティのある一次情報を伝える
- メリットだけでなくデメリットも正直に伝える
- 複数の商品をおすすめする場合は、おすすめNo.1を明確にする
